## 問題 2：PAYLOAD_DATABASE_PROVIDER 設定失敗（production 用 SQLite）

**症狀：** Production 的 `/api/newsletter` 和 `/admin` 回傳 500。本機正常。

**原因：** 用 `echo "postgres" | vercel env add PAYLOAD_DATABASE_PROVIDER production` 設定環境變數，echo 帶入了換行符，導致實際值為 `'postgres\n'`。

程式碼中的比較：
```ts
const isProd = process.env.PAYLOAD_DATABASE_PROVIDER === 'postgres'
// 'postgres\n' === 'postgres' → false → 用 SQLite
```

Production serverless 環境沒有持久化的本地檔案系統，SQLite 寫入失敗 → 500。

**解法：用 Vercel REST API 直接寫入（無 echo pipe）**

```bash
VERCEL_TOKEN=$(cat "$HOME/Library/Application Support/com.vercel.cli/auth.json" | python3 -c "import sys,json; print(json.load(sys.stdin)['token'])")
PROJECT_ID=$(cat .vercel/project.json | python3 -c "import sys,json; print(json.load(sys.stdin)['projectId'])")
TEAM_ID=$(cat .vercel/project.json | python3 -c "import sys,json; print(json.load(sys.stdin)['orgId'])")

vercel env rm PAYLOAD_DATABASE_PROVIDER production --yes
vercel env rm PAYLOAD_DATABASE_PUSH production --yes

curl -s -X POST "https://api.vercel.com/v10/projects/$PROJECT_ID/env?teamId=$TEAM_ID" \
  -H "Authorization: Bearer $VERCEL_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"key":"PAYLOAD_DATABASE_PROVIDER","value":"postgres","type":"plain","target":["production"]}'

curl -s -X POST "https://api.vercel.com/v10/projects/$PROJECT_ID/env?teamId=$TEAM_ID" \
  -H "Authorization: Bearer $VERCEL_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"key":"PAYLOAD_DATABASE_PUSH","value":"true","type":"plain","target":["production"]}'
```

**關鍵：** token 路徑是 macOS 的 `$HOME/Library/Application Support/com.vercel.cli/auth.json`，不是 `~/.config/vercel/auth.json`。

## 問題 4：Payload Admin 500（Payload v3.84.1 缺少 RootLayout）

**症狀：** 訪問 `/admin` 或 `/admin/login` 時出現 client-side error：
```
TypeError: Cannot destructure property 'config' of undefined
```

**原因：** Payload v3.84.1 把所有 admin UI providers（`useConfig()`、`useTheme()` 等）移到 `RootLayout` 中。缺少 `app/(payload)/admin/layout.tsx` 時，這些 hook 拿到 `undefined`，CodeEditor 等元件初始化就崩潰。

**解法：** 新增 `src/app/(payload)/admin/layout.tsx`：

```tsx
import { RootLayout, handleServerFunctions } from '@payloadcms/next/layouts'
import React from 'react'
import { importMap } from './importMap'
import configPromise from '@payload-config'
import type { ServerFunctionClient } from 'payload'

type Args = { children: React.ReactNode }

const serverFunction: ServerFunctionClient = async function (args) {
  'use server'
  return handleServerFunctions({ ...args, config: configPromise, importMap })
}

export default async function AdminLayout({ children }: Args) {
  return (
    <RootLayout config={configPromise} importMap={importMap} serverFunction={serverFunction}>
      {children}
    </RootLayout>
  )
}
```

**注意：** `serverFunction` 必須包在 async function 並加 `'use server'`，直接傳 `handleServerFunctions` 會有 TypeScript 型別錯誤（`ServerFunctionHandler` 不可賦值給 `ServerFunctionClient`）。

## 問題 5：Admin 帳號密碼驗證失敗

**症狀：** 透過 `/api/users` POST 建立 `doyen@opand.com` 帳號後，登入時回傳「incorrect email or password」。

**原因：** Payload 的密碼 hash 格式為 `crypto.pbkdf2(password, salt, 25000, 512, 'sha256')`，透過 REST API 建立的帳號 hash 格式不一致。

**解法：** 直接計算正確 hash 並更新 Neon DB 的 `users` table（`hash` 和 `salt` 欄位）。

## 問題 6：課程詳情頁無內容

**症狀：** `/courses/[slug]` 顯示「課程內容即將上架，敬請期待」，而非實際課程內容。

**原因：** `courses.body` 欄位存的是 Lexical JSON（JSONB），頁面元件沒有渲染它。

**解法：** 在 `src/app/(frontend)/courses/[slug]/page.tsx` 加入 `RichText`：

```tsx
import { RichText } from '@payloadcms/richtext-lexical/react'

{course.body ? (
  <div className="prose prose-lg max-w-none mb-12">
    <RichText data={course.body as Parameters<typeof RichText>[0]['data']} />
  </div>
) : null}
```

Lexical JSON 結構（seed 時手動建立）：
```js
const para = (text) => ({
  type: 'paragraph', format: '', indent: 0, version: 1, direction: 'ltr',
  children: [{ type: 'text', format: 0, style: '', detail: 0, mode: 'normal', text, version: 1 }]
})
const body = (...nodes) => ({ root: { type: 'root', format: '', indent: 0, version: 1, direction: 'ltr', children: nodes } })
```

| 項目 | aiart.tw | ai-coach-template |

1. `gh repo clone OvermanAI/ai-coach-template`

## 經驗總結

1. **`echo pipe` 絕對不能用於 `vercel env add`** — 換行符問題每次都會出現，必須用 REST API。
2. **Payload v3.84.1+ 必須有 `admin/layout.tsx`** — 沒有這個檔案，admin 頁面全部崩潰。
3. **Vercel auth token 在 macOS 的路徑** 是 `$HOME/Library/Application Support/com.vercel.cli/auth.json`，不是 Linux 的 `~/.config/vercel/auth.json`。
4. **`npm install` 必須加 `--legacy-peer-deps`** — Payload v3.84 + Next.js 15.5 有 peer dep 衝突，`.npmrc` 已寫死此設定。
5. **`npm run seed` 有 ESM 相容性問題** — `@next/env` 在 tsx 環境下的 default import 為 undefined。目前 workaround 是直接用 Neon + REST API seed，此問題在 template 中已記錄待修。
